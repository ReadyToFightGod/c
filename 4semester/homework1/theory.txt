1) Для чего используются системы контроля версий?
Система контроля версий — это система, записывающая 
изменения в файл и позволяющая вернуться позже к 
определённой версии.
СКВ используются для того, чтобы записывать и 
отслеживать изменения в файлах с возможностью 
откатиться к последней проверенной (корректно
работающей) версии.

2) Из каких основных действий состоит 
взаимодействие с Git?
git commit - сохранение текущего состояния репозитория
(папочки, за которой следит гит)
git stage - выбор файлов из репы, которые надо загрузить
на сервер.
git ignore - указание на файлы, которые надо игнорировать
git push - загрузка репы на сервер
git pull - скачивание репы с сервера
git branch - показ текущей ветки (последовательности 
коммитов)
git checkout - перемещение между разными ветками


3) Какие типы контейнеров Вам известны? 
Классифицируйте все имеющиеся контейнеры STL и 
их адаптеры.
Есть 2 бошьших класса контейнеров: гомогенные
(для 1 типа данных) и гетерогенные (для многих)
Гомогенные:
а) основаны на массиве - элементы лежат строго 
по порядку - vector, array, deque
b) основаны на дереве - set, multiset /// map, multimap 
(время поиска быстре, хранится массив пар - аналог dict в питоне)
с) узловые - list, forwarded_list (односвязный список)
Гетерогенные (pair, tuple)
d) хеш-таблица — неотсортированы, но работают быстрее - то же, что и в 2) но спереди unordered_
e) адаптеры: stack, queue, priority_queue (отсортированная очередь)
