1) В чем заключается концепция встраивания вызовов функции?
Процесс встраивания состоит в замене вызова функции на тело этой функции. Т.е. функция вынесена отдельно, но в процессе 
выполнения программы ее тело как бы встраивается в основной код, и тогда не теряется память на стек отдельной функции.

2) В каких случаях используется ключевое слово auto?
Ключевое слово auto говорит, что компилятор на этапе компиляции должен определить тип переменной
(или функции) на основе типа инициализируемого выражения. Auto используют, если не хочется писать длинные типы данных при 
инициализации (std::vector, к примеру) или тип переменной заранее неизвестен.

3) Какие аргументы функции могут иметь значения по умолчанию?
Вообще, все аргументы могут иметь значения по умолчанию, но есть ограничения по тому, как их писать.
Все аргументы, имеющие значение по умолчанию, должны стоять после тех, у которых такого значения нет.
(и порядок записи аргументов со значением такой, что первее стоят те, которые пользователь вероятнее
всего опустит.)

4) Что такое перегрузка функций?
Перегрузка функций – это создание нескольких функций с одним именем, но с разными параметрами. Например,
при создании функции сложения двух чисел, необходимо учитывать, что тип аргументов может быть разным
(и тип вывода тоже), поэтому можно создать несколько функций с одним именем, а параметры указать разные.

5) Как передать функцию в функцию? Как сохранить лямбда-функцию?
Можно передать функцию по константной ссылке, для этого нужно обернуть ее в
шаблон std::function<тип_функции(типы_всех_переменных)> "функция" (нужно #include<functional>) 
Сохранить лямбда-функцию можно с помощью этого же шаблона std::function<что-то-там-про-типы> smth = [](){}

6) Что такое ссылка? Чем отличается передача аргументов в функцию по ссылке и по значению? 
Как принято передавать массивы и другие контейнеры в функции? Может ли ссылка быть константной? Что это означает?
Ссылка - это обертка на указатель (просто отдельная переменная, в которой содержится указатель). При передаче по
аргументов по ссылке их значения можно изменять глобально (т.е. если в функции аргумент стал равен 5, то во
всей программе он теперь будет равен 5). Если надо провести манипуляции с содержимым переданного аргумента,
но исходноая переменная не должна измениться, то передать надо по значению (т.к аргументы просто копируются в функцию).
Массивы и другие контейнеры принято передавать по ссылке (обычной или константной), потому что это быстрее и
эффективнее. Константная ссылка - это ссылка на переменную, значение которой нельзя изменить через эту же ссылку.
